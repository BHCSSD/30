# 4.2.3 List and String Similarities

## Indexing

Use **indexing** to access a specific element in a sequence.

### Standard Indexing (Starts at 0)

* A list of length `N` has indexes from `0` to `N-1`

  ```python
  L = [42, True, 'wahoo']
  print(L[0])  # 42
  print(L[1])  # True
  print(L[2])  # 'wahoo'
  ```

> [!WARNING] 
> Accessing an index **outside the list** range will cause an `IndexError`
> 
> ```python
> L = [42, True, 'wahoo']
> print(L[3]) # Error: list index out of range
> ```
> 

### Negative Indexing (Counts from End)

* Negative indexes count from the end of the sequence, starting at $-1$

  ```python
  L = [42, True, 'wahoo']
  print(L[-1])  # 'wahoo'
  print(L[-2])  # True
  print(L[-3])  # 42
  ```

-----

## IMPORTANT!!!!i! Slicing

* **Syntax:** `L[start:end]` returns a **new sublist**

  * The **end index is not included** (it is *exclusive*)

  ```python
  L = ['a', 'b', 'c', 'd', 'e', 'f']
  
  print(L[2:5])     #['c','d','e']
  print(L[:4])      #['a','b','c','d']      (Starts at index 0)
  print(L[4:])      #['e','f']              (Goes until the end)
  print(L[0:5:2])   #['a','c','e']          (Step/skip count of 2)
  print(L[::-1])    #['f','e','d','c','b','a'] (Reverses the list)
  print(L[:11])     #[]                    (Empty slice)
  ```

* Slicing **never crashes**, even if indexes are out of range:

  ```python
  print(L[4:8])  # ['e','f'] (Stops at the end of the list)
  print(L[6:])   # []        (Starts past the end, returns empty)
  ```

  <details>
  <summary>Answer</summary>

    * **C. `L[-2]`**: Counts two from the end, which is the value **4**
    * **D. `L[-3]`**: Counts three from the end, which is the value **3**
    * **E. `L[0] + L[1]`**: $1 + 2$ is **3**

## Two Types of For Loops

### 1. Looping by Index  (for i in range() )

* You loop over the **indexes** using `range( len(L) )`  

  ```python
  L = [42, True, 'wahoo']
  for i in range( len(L) ):
      print(L[i])
  # Output:
  # 42
  # True
  # wahoo
  ```

### 2. Looping by Value (for each in)

* You loop **directly through the values** themselves

  ```python
  L = [42, True, 'wahoo']
  for value in L:
      print(value)
  # Output:
  # 42
  # True
  # wahoo
  ```

1. For what list `L` will the following code print **3**? *HINT* `v%2==0` is checking to see if `v` is even

    ```python
    t = 0
    for v in L:
        if (v % 2 == 0):
            t += 1
    print(t)
    ```

* A.  `L = list(range(5, 10))`
* B.  `L = list(range(2, 8))`
* C.  `L = [3, 2, 3, 2, 3]`
* D.  `L = [2, 3, 2, 3]`

  <details>
  <summary>Answer</summary>

  **B. `L = list(range(2, 8))`**.

    * a.  `[5,6,7,8,9]`: Even values are `6,8`. Count is **2**
    * b.  `[2,3,4,5,6,7]`: Even values are `2,4,6`. Count is **3**
    * c.  `[3,2,3,2,3]`: Even values are `2,2`. Count is **2**
    * d.  `[2,3,2,3]`: Even values are `2,2`. Count is **2**

  </details>



2. For what list `L` will the following code **NOT print "yes"**? 

    ```python
    m = 'yes'
    for i in range(len(L)):
        if (L[i] != i):
            m = 'no'
    print(m)
    ```

* A.  `L = [0,1]`
* B.  `L = []`
* C.  `L = list(range(10))`
* B.  `L = [1,2, 3]`

  <details>
  <summary>Answer</summary>

  **D. `L = [1, 2, 3]`**.

    * The code prints "yes" only if `L[i] == i` for *all* valid indexes i
    * a.  `L = [0, 1]`: `L[0]` is 0, `L[1]` is 1. All match. Prints "yes"
    * b.  `L = []`: The loop `range(len(L))` is `range(0)`, so the loop body never runs. `m` stays `'yes'`. Prints "yes"
    * c.  `L = list(range(10))`: This list is `[0, 1, 2, ..., 9]`. `L[i]` is always equal to i. Prints "yes".
    * d.  `L = [1, 2, 3]`: `L[0]` is 1, which is **not** 0. The `if` condition is true, `m` is set to `'no'`. Prints **"no"**

  </details>
