## **4.2.8 List Methods**

List methods are divided into two categories: 
* **Non-Mutating** (they just give you info)
* **Mutating** (they change the list itself)

---

### **Non-Mutating List Methods**

These methods do **not change** the original list. They are safe to use when you just need to search data.

#### `index()` and `count()`

* `L.index(v)` Returns the **first index** where value `v` appears.
* `L.count(v)` Returns **how many times** `v` appears in the list.

  ```python
  L = [5, 3, 8, 7, 8]
  print(L.count(8))  # 2
  print(L.index(8))  # 2 (finds the FIRST one)

  ```

> [!WARNING]
> If `v` is **not found**, `L.index(v)` will **crash** your program with a `ValueError`.

**How to safely find an index:**

```python
# Option 1: "Look Before You Leap" 
if v in L:
    print(L.index(v))

# Option 2: Write a custom helper function
# aka linear search
def listFind(L, v):
    for i in range(len(L)):
        if L[i] == v:
            return i
    return -1

```

---

### **Mutating List Methods**

These methods **change the list directly**.

> [!IMPORTANT]
> Most mutating methods return `None`.
> * If you try to do `L = L.append(5)`, you will accidentally destroy your list and turn it into `None`
>
> **REMIDNER Method vs. Function:**
> * L.sort() is a method (modifies the list in place)
> * sorted(L) is a function (creates a copy)

#### Adding Elements: `append`, `extend`, `insert`

* `L.append(v)`: Adds a single value `v` to the **very end**
* `L.extend(M)`: Takes all values from list `M` and adds them to the end of `L`
* `L.insert(i, v)`: Squeezes value `v` into **index `i`**, shifting everything else to the right

  ```python
  L = [2, 5, 3]
  L.append(4)       # [2, 5, 3, 4]
  L.extend([7, 8])  # [2, 5, 3, 4, 7, 8]
  L.insert(1, "HI") # [2, "HI", 5, 3, 4, 7, 8]

  ```

#### Removing Elements: `remove` and `pop`

* `L.remove(v)`: Searches for the **first occurrence** of value `v` and deletes it
  * This crashes if `v` isn't there
* `L.pop(i)`: Removes the value at index `i` AND **returns it** so you can use it
* `L.pop()`: (No index) Removes and returns the **last value**

  ```python
  L = [10, 20, 30]
  val = L.pop(1) # Removes 20
  print(val)     # 20
  print(L)       # [10, 30]

  ```

### **Checkpoint**

1. Which of the following is a **non-mutating** list method?

* A. `L.sort()`
* B. `L.extend(M)`
* C. `L.index(v)`
* D. `L.pop(i)`

  <details>
  <summary><b>Answer</b></summary>
  <blockquote>
  <b>C. <code>L.index(v)</code></b>

  <code>sort()</code>, <code>extend()</code>, and <code>pop()</code> all modify the list. <code>index()</code> just looks at the origional list and returns a number without changing anything.
  </blockquote>
  </details>

2. Which of the following **mutating** methods returns a value that is **not None**?

* A. `L.pop(i)`
* B. `L.append(v)`
* C. `L.insert(i, v)`
* D. `L.remove(v)`

  <details>
  <summary><b>Answer</b></summary>
  <blockquote>
  <b>A. <code>L.pop(i)</code></b>

  <code>append</code>, <code>insert</code>, and <code>remove</code> return <code>None</code>. <code>pop</code> is the unique one that returns the item it just removed, which is very useful for "grabbing" an item out of a list.
  </blockquote>
  </details>

3. Which of the following is **NOT** a standard way to deal with the fact that Python lists lack `L.find(v)`?

* A. Write your own `listFind(L, v)`
* B. Use `L.index(v)` but first check that `v in L`
* C. Convert list to a string and use `s.find(v)`
* D. Use a `for` loop to check manually

  <details>
  <summary><b>Answer</b></summary>
  <blockquote>
  <b>C. Convert list to a string and use <code>s.find(v)</code></b>

  Converting a list to a string (like <code>str([1, 2, 3])</code>) results in <code>"[1, 2, 3]"</code>. Searching that string will look for characters like commas and brackets, and it won't give you the correct numerical index of the list item!
  </blockquote>
  </details>
